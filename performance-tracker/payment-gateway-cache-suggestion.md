# 支付网关 PR#4 评审缓存建议

> 源文件：agentraft-doe-source / PR#4 "接入新支付网关" 评审评论

> 提取角色：代码评审协调员

## 建议摘要

在支付网关接入 PR 代码评审中，建议引入缓存层以减少重复支付查询对下游系统的压力。

### 当前问题

- `ProcessPayment` 处理中未对商户/订单信息做缓存
- 重复回调场景下，每次均穿透到下游，存在性能风险
- 压测结论 DINT-LOADTEST-P95 延迟低于 180ms 虽达标，但增加缓存可进一步提升余量

### 缓存建议

1. **商户配置缓存**：将商户密钥、回调地址等配置数据缓存至本地内存（TTL 300s），避免每次请求读取外部配置存储
2. **幂等 Key 缓存**：对已处理的幂等 Key 短时缓存（TTL 60s），防止重复回调重复处理
3. **网关令牌缓存**：网关鉴权 token 缓存至过期前 30s 刷新，减少认证请求

### 预期收益

- 降低下游系统调用量约 40%
- P95 延迟可进一步降低至 100ms 以下
- 提升网关在峰值期的吞吐能力
